# 绳墨 - 集成AI游戏工具

**绳墨** 是一款专为个人打造的、功能丰富、界面简约的集成AI游戏工具。它以浏览器为载体，结合强大的后端服务，为您提供一个高度可定制化、私密且高效的 AI 体验环境。无论您是想进行沉浸式角色扮演、利用 AI 进行绘图创作，还是管理复杂的对话场景、阅读并引用参考资料，绳墨都能满足您的需求。

---

## ✨ 核心特性

*   **🚀 双 AI 引擎驱动:** 同时集成强大的 **谷歌 Gemini** 模型和 **NovelAI** 图像生成能力。
*   **🎭 深度角色扮演:** 支持多角色定义（永久/临时）、状态管理（默/活/用/更）、记忆与设定分离、角色可见性控制，构建生动的虚拟世界。
*   **🎨 AI 绘图集成:** 无缝调用 NovelAI 进行图像生成，支持丰富的参数调整和角色绘图模板。
*   **📚 小说阅读与引用:** 内建小说阅读器，可将多部激活的小说内容作为动态参考信息（`{{原著小说}}`）注入 AI 对话。
*   **🛠️ 智能工具箱:** 配备多种专用 AI 工具（绘图大师、游戏主持、写作大师、角色更新大师、私人助理），拓展应用场景。
*   **💬 多聊天室管理:** 支持创建、管理、重命名、删除、导入/导出多个独立的聊天场景，保持对话隔离。
*   **⚙️ 高度可配置:**
    *   全局 AI 参数、提示词预设、工具默认配置。
    *   支持 **聊天室级别** 的配置覆盖，可为特定聊天室定制通用 AI 参数及各工具的行为。
*   **📱 移动端优化:** 主要界面为移动端交互设计，布局紧凑，操作便捷，支持长按操作和全屏模式。
*   **🔒 本地化部署:** 所有数据（配置、聊天记录、角色、小说、图片）均存储在本地，确保隐私安全。
*   **📦 便捷备份恢复:** 支持一键导出/导入完整配置和单个聊天室数据（ZIP 格式），以及单个角色（JSON）和提示词预设（JSON）。

## 📝 详细功能介绍

### 🤖 AI 交互核心

*   **Gemini 模型代理:**
    *   支持接入多种谷歌 Gemini 模型进行文本生成。
    *   可精细调整全局生成参数：温度（Temperature）、顶P（Top P）、顶K（Top K）、最大输出令牌数。
    *   默认使用 `application/json` 响应格式，并支持自定义 Schema 解析脚本（JavaScript）对 AI 输出进行预处理。
    *   支持配置 JSON Schema 以获取结构化输出。
    *   支持全局系统指令（System Instruction）和多轮对话提示词预设（Prompt Preset Turns）。
    *   提供多个 API 密钥轮询机制，自动跳过暂时不可用的密钥，并记录失败次数，提高可用性。
    *   默认关闭 Gemini 的所有安全设置，提供更自由的交互体验。
*   **NovelAI 绘图代理:**
    *   集成 NovelAI 图像生成接口 (`/novelai-proxy`)。
    *   可配置使用的 NovelAI 模型（如 NAI Diffusion 3/4 等）。
    *   支持设置默认的正面和负面提示词。
    *   支持艺术家风格链（Artist Chain）。
    *   可为每个角色设定独立的绘图模板（Drawing Template），自动应用于绘图提示词。
    *   全面控制生成参数：图像宽度、高度、步数（Steps）、引导系数（Scale）、CFG 重调（CFG Rescale）、采样器（Sampler）、噪点调度（Noise Schedule）、种子（Seed，0 为随机）。
    *   自动处理 API 请求与响应，将生成的图片直接显示在聊天界面，并提供设为背景/下载/重绘操作。
    *   内置请求队列，避免并发问题。

### 💬 聊天室系统

*   **多聊天室:**
    *   创建和管理多个独立的聊天室 (`chatrooms` 目录)，每个聊天室拥有自己的配置、历史记录、角色和小说。
    *   支持聊天室的重命名、删除。
    *   可方便地切换当前活动的聊天室。
*   **独立配置:**
    *   每个聊天室可设置独特的“扮演规则”（Roleplay Rules）和“公共信息”（Public Info），这些信息可作为上下文注入 AI 请求。
    *   支持为聊天室设置独立的背景图片。
    *   支持为聊天室配置 **覆盖设置 (Override Settings)**，可以覆盖全局的通用 AI 参数和各个工具的特定参数（模型、提示词、数据库、Schema 等）。
*   **持久化存储:**
    *   聊天记录自动保存在本地对应的聊天室文件夹内 (`history.json`)。
    *   聊天室特定配置（如角色状态、激活的小说、小说阅读进度、背景图、覆盖设置、角色可见性等）保存在 `chatroom_config.json` 中。
*   **导入与导出:**
    *   支持将单个聊天室的所有数据（配置、历史、角色、小说、背景图）导出为 ZIP 压缩包。
    *   支持从 ZIP 压缩包导入聊天室数据，轻松迁移或分享场景（名称冲突会自动重命名）。

### 🎭 角色扮演（RP）系统

*   **角色定义:**
    *   在每个聊天室中，可以创建和管理多个“永久角色”（存储在 `roles` 子目录下的 JSON 文件）。
    *   每个角色包含独立的“设定”（Setting）、“记忆”（Memory）和“绘图模板”（Drawing Template）。
    *   支持角色的编辑、重命名、删除，以及单个角色的导入/导出（JSON 格式）。
*   **临时角色:**
    *   支持在聊天过程中动态创建“临时角色”，无需预先定义文件，其状态直接保存在 `chatroom_config.json` 中。
*   **角色状态管理:**
    *   提供直观的角色状态切换按钮，管理每个角色的行为模式：
        *   `默` (默认/静默): 角色不主动参与，其设定和状态仍可作为上下文信息。
        *   `活` (活动): 角色由 AI 控制，根据设定和场景自动发言或行动。
        *   `用` (用户控制): 角色行为由用户手动输入。当 AI 指定此角色行动时，会弹出 `💬` 按钮提示用户输入。
        *   `更` (更新): 触发 **角色更新大师** 工具（如果已启用），分析近期对话以更新该角色的记忆或设定。
*   **角色可见性:**
    *   可以单独设置每个 **永久角色** 在角色状态控制面板中是否可见（通过长按 `默` 按钮或在设置中勾选）。隐藏的角色不会出现在控制面板中，但其数据和状态依然存在。
*   **角色状态存储:**
    *   基础状态（默/活/用）和可见性存储在 `chatroom_config.json` 的 `roleStates` 和 `roleVisibility` 中。
    *   “角色详细状态”（Role Detailed States）存储在 `chatroom_config.json` 的 `roleDetailedStates` 中，用于存储更复杂的、通常由 **游戏主持人** 更新的角色信息（如姿态、装备、状态效果等）。

### 🛠️ 智能工具箱

绳墨内置了一系列专用 AI 工具，可在后台运行，辅助主对话流程：

*   **通用配置:** 每个工具均可独立启用/禁用，并可配置其使用的 Gemini 模型、主提示词、数据库指令、响应 Schema 及解析脚本。这些配置可以在全局设置，也可以被聊天室的覆盖设置所替代。
*   **绘图大师 (Drawing Master):**
    *   负责解析绘图需求，结合角色绘图模板和场景信息，生成发送给 NovelAI 的具体提示词。响应结果直接触发 NovelAI 绘图流程。
*   **游戏主持人 (Game Host):**
    *   能够理解并执行添加/删除临时角色、静默/激活永久角色的指令。
    *   能够输出场景上下文（时间、地点、角色位置等）、更新角色详细状态（姿态、装备、状态等），并将这些信息结构化地展示在聊天界面。
*   **写作大师 (Writing Master):**
    *   用于根据对话内容或特定指令，辅助生成描述性文本或其他创意内容。
*   **角色更新大师 (Character Update Master):**
    *   分析最近的对话历史，智能地更新指定角色的“记忆”或“设定”文件内容（仅限永久角色）。输出结果提供“保存”按钮以应用更改。
*   **私人助理 (Private Assistant):**
    *   一个通用的后台工具，可以用于执行不需要特定格式输出的通用任务或进行自由交流。

### 📚 小说阅读与参考

*   **小说管理:**
    *   支持在每个聊天室中添加和管理多部小说（存储在 `novels` 子目录下的 JSON 文件）。
    *   提供从剪贴板粘贴文本创建新小说的功能，后端会自动将长文本分割成合适的段落（Segments）。
    *   自动提取文本中的章节标记（如“第一章”），并能基于段落数量自动生成补充目录（TOC）。
    *   支持小说的重命名和删除。
*   **小说阅读器:**
    *   内建一个独立的小说阅读界面。
    *   提供“书架”视图，展示当前聊天室的所有小说，并允许选择当前阅读的小说和激活/取消激活小说。
    *   提供“目录”视图，方便在章节间快速跳转。
    *   自动记忆每部小说上次阅读到的滚动位置，并将其存储在 `chatroom_config.json` 的 `novelCurrentSegmentIds` 中。
*   **激活与引用:**
    *   用户可以在书架中勾选要“激活”的小说。
    *   所有被激活的小说内容（根据各自当前的阅读位置和全局配置的“原著小说字符数”）会自动提取并组合，作为一个特殊的上下文信息（`{{原著小说}}`）提供给 AI，使其能够“阅读”并引用相关内容。

### ⚙️ 配置与管理

*   **全局配置:**
    *   所有全局设置（如默认 AI 参数、API Key 相关设置、NovelAI 设置、聊天室排序、工具全局配置等）保存在根目录的 `config.json` 文件中。
    *   前端设置更改后会自动通过 `/autosave-config` 接口保存到后端。
*   **提示词预设:**
    *   支持配置一套包含系统指令和多轮用户/模型对话的模板。
    *   这些预设会作为基础上下文添加到发送给 Gemini 的请求中。
    *   支持将预设配置（V2 格式，包含系统指令、轮次、共享角色设置、工具设置）导出为 JSON 文件，或从文件中导入。
*   **运行时错误日志:**
    *   前端界面会捕获并显示 JavaScript 运行时错误，方便调试。
*   **完整备份与恢复:**
    *   提供一键导出整个应用状态（全局配置 + 所有聊天室数据）为 ZIP 文件的功能 (`/export-full-config-zip`)。
    *   支持从 ZIP 文件恢复整个应用状态 (`/import-full-config-zip`)，会覆盖现有数据。
*   **数据清理:**
    *   提供“清空全部配置”的危险操作选项 (`/clear-all-config`)，用于彻底重置应用（删除 `config.json`，清空 `chatrooms` 和 `images/generated` 目录）。

### ✨ 用户界面与体验

*   **界面设计:**
    *   主聊天界面 (`gemini_chat.html`) 优先考虑移动设备，布局紧凑，元素间距小。
    *   采用暗色主题，适合长时间使用。
    *   提供一个独立的编辑器界面 (`editor.html`)，用于项目文件的直接编辑（详见作者的话）。
*   **设置面板:**
    *   通过点击 `⚙️` 图标打开，支持层级导航和返回。
    *   设置项按功能模块清晰分类，包括聊天室特定的覆盖配置。
*   **聊天界面:**
    *   消息气泡样式区分用户和 AI。
    *   AI 消息气泡上方显示角色标识按钮。
    *   支持显示由 NovelAI 生成的图片，点击可全屏查看。
    *   游戏主持人的输出有定制化的卡片式展示格式，支持切换视图。
    *   提供消息操作菜单（**短按** 消息旁的头像/图标触发），支持：
        *   `🔄` 切换显示 AI 消息的原始 JSON 数据或格式化文本。
        *   `🖼️` 将 NovelAI 图片设为聊天室背景（短按）或下载图片（长按）。
        *   `🖌️` 重新触发绘图（针对 NovelAI 图片消息）。
        *   `💾` 保存角色更新（针对角色更新大师消息）。
        *   `✕` 删除单条消息（短按） 或 删除此条及以下所有消息（长按）。
    *   支持直接编辑已发送的消息内容（**长按** 消息旁的头像/图标触发）。
*   **交互元素:**
    *   顶栏提供设置入口、小说阅读器入口、运行/暂停按钮、角色列表开关、管理员消息/临时角色添加按钮。
    *   角色列表面板可方便地查看和切换角色状态（默/活/用），并提供更新（更）触发和临时角色删除（长按默）功能。
    *   提供加载状态指示器（Spinner）和请求重试时的视觉提示。
    *   为用户控制角色 (`用` 状态) 提供待处理动作提示按钮 (`💬`)。

## 🧭 绳墨 - 详尽操作指南

本指南将引导您全面了解并熟练使用 **绳墨** 的各项功能。

---

### 🚀 快速开始

1.  **启动后端:** 在您的终端或服务器环境中，运行 `start_backend.bat` 或 `start_backend.sh` 启动 Python 后端服务。
2.  **访问前端:** 在您的浏览器（移动端推荐 Edge 或 Chrome 系浏览器）中打开指向 `gemini_chat.html` 的地址（通常是 `http://<您的IP地址(本地是：127.0.0.1)>:8888`）。
3.  **初始配置:** 首次使用时，强烈建议先进入设置面板（点击左上角 `⚙️` 图标），在 **API 设置** 页面配置您的谷歌 Gemini API 密钥，并点击 **拉取模型列表**。如果需要绘图功能，请在 **绘图设置** 页面配置您的 NovelAI API 密钥。

---

### 💬 主聊天界面 (`gemini_chat.html`)

这是您与 AI 交互的主要场所。

#### 顶栏 (Top Toolbar)

位于界面最上方，固定显示，包含核心控制按钮：

*   **`⚙️` 设置图标 (左上角):**
    *   **短按:** 打开/关闭 **设置面板**。所有全局配置、聊天室管理、角色/小说编辑等都在这里进行。
    *   **长按:** 切换浏览器 **全屏模式**。
*   **`📖` 小说按钮 (右上角):**
    *   **点击:** 打开/关闭 **小说阅读界面**。您可以在此阅读、管理当前聊天室关联的小说，并将其内容作为参考信息提供给 AI。
*   **`👤` 角色按钮 (顶栏左二):**
    *   **短按:** 打开/关闭 **角色状态控制面板**（显示在顶栏下方）。
    *   **长按:** 直接跳转到 **设置面板** 中的 **聊天室角色列表** 页面，方便管理角色文件。
*   **`📜` 规则按钮 (顶栏左三):**
    *   **点击:** 直接跳转到 **设置面板** 中的 **当前聊天室设置** 页面，方便查看和编辑扮演规则、公共信息等。
*   **加载指示器 (旋转圆圈，顶栏中央):**
    *   当 AI 正在处理请求时，会显示并旋转。
    *   如果请求失败并尝试使用下一个 API 密钥重试时，圆圈顶部会短暂变为 **红色**。
*   **`📏` 管理员按钮 (顶栏右侧，运行按钮左侧):**
    *   **短按:** 在聊天区域 **创建一条新的“管理员”消息**，并立即进入编辑模式，允许您输入旁白、指令或场景描述等非角色发言。
    *   **长按:** 弹出输入框，允许您快速 **添加一个新的“临时角色”** 到当前聊天室。
*   **`▶`/`◼` 运行/暂停按钮 (顶栏右侧):**
    *   **点击:** 切换 AI 的 **自动运行状态**。
        *   `▶` (暂停状态): AI 不会自动响应。点击后切换到 `◼` 运行状态。
        *   `◼` (运行状态): AI 会根据角色状态和设定自动响应。点击后切换到 `▶` 暂停状态。
    *   **解除暂停 (`▶` -> `◼`) 时的行为:**
        *   如果当前聊天室只有一个角色状态为 `活`，会自动触发该角色响应。
        *   如果有多个角色状态为 `活`，会在界面右上方弹出一个 **活动角色触发列表**，让您选择首先由哪个角色响应。
        *   如果 **游戏主持人** 工具已启用，会自动触发一次游戏主持人的响应。
        *   如果上一条消息是用户输入，且内容包含特定感官词（看、听、闻、模、尝等），且 **写作大师** 工具已启用，会自动触发一次写作大师的响应。
    *   **长按 (`▶` 或 `◼` 状态下均可):** 如果 **私人助理** 工具已启用，会直接触发一次私人助理的响应。
*   **`💬` 待处理动作按钮 (顶栏右下角，仅特定情况出现):**
    *   当某个角色状态为 `用` (用户控制)，并且轮到该角色行动时（例如由 AI 指定 `nextRoleToAct`），此按钮会出现。
    *   **点击:** 为该角色 **创建一条新消息** 并进入编辑模式，让您手动输入该角色的发言或行动。点击后此按钮消失。

#### 角色状态控制面板 (`#role-buttons-wrapper`)

通过点击 `👤` 角色按钮显示/隐藏。

*   **布局:** 每个可见的 **永久角色** 和所有 **临时角色** 一行，左侧是代表角色的主按钮，右侧是状态选择按钮（默认隐藏）。
*   **角色主按钮 (`[角色名首字母/符号]`):**
    *   显示角色的简写标识（通常是名称首字母，为避免重复可能会使用其他字符或数字）。
    *   **临时角色** 的按钮背景色为白色，以作区分。
    *   **短按:** 展开/收起右侧的 **状态选择按钮**。
    *   **长按:** 直接跳转到 **设置面板** 中该角色的 **角色详情** 页面。
*   **状态选择按钮 (`默`, `活`, `用`, `更` - 点击角色主按钮后出现):**
    *   **含义:**
        *   `默` (默认/静默): 角色不主动行动，其设定和状态仍可作为上下文信息。
        *   `活` (活动): 角色由 AI 控制，在轮到它或被触发时会自动生成响应。
        *   `用` (用户控制): 角色行为需要用户手动输入。当 AI 指定此角色行动时，会弹出 `💬` 按钮提示用户输入。
        *   `更` (更新): **点击** 此状态会触发 **角色更新大师** 工具（如果已启用），该工具会分析近期对话，尝试更新此角色的记忆或设定。**注意:** 这不是一个持久状态，点击后会立即触发更新动作，角色状态通常不会停留在 `更`。
    *   **短按:** 将角色切换到对应的状态。当前状态的按钮会有高亮显示。状态更改会自动保存。
    *   **特殊长按:**
        *   **长按 `用` 按钮:** 立即为该角色 **创建一条新消息** 并进入编辑模式，相当于 `💬` 按钮的功能。
        *   **长按 `默` 按钮 (仅对临时角色有效):** 弹出确认框，确认后 **删除该临时角色**。
        *   **长按 `默` 按钮 (仅对永久角色有效):** 将该角色 **隐藏**，使其不再显示在此面板中（可在设置中恢复）。

#### 聊天区域 (`#chat-area`)

显示对话消息的主体区域。

*   **消息气泡:**
    *   不同样式区分用户发言 (`user-message`) 和 AI 发言 (`ai-response`)。
    *   AI 发言气泡上方会显示代表该 AI 角色的 **角色名称按钮** (短按触发菜单，长按触发编辑)。
*   **消息内容:**
    *   **普通文本:** 显示角色的发言或行动描述。
    *   **图片:** 如果是 NovelAI 生成的图片，会直接显示图片。点击图片可 **全屏查看**。
    *   **游戏主持:** 以特定格式显示场景时间、地点、角色位置、角色状态等信息，并提供视图切换按钮（`🕒 时间`, `📍 地点`, `👤 角色`）。
    *   **角色更新大师:** 显示更新后的角色记忆和设定摘要。
    *   **其他工具:** 根据其配置和功能显示相应输出。

#### 图片查看器 (`#image-viewer-page`)

*   当您点击聊天气泡中的图片时，会覆盖整个屏幕显示该图片。
*   **点击图片查看器任意位置:** 关闭查看器，返回聊天界面。

2.  **提示词预设 (`prompt-preset-page`)**

    *   **上次发送的主要绘图提示词:** 只读显示。
    *   **参数 (Width, Height, Steps, Scale, CFG Rescale, Seed):** 输入框。**修改后自动保存**。

*   **触发:** **短按** 消息气泡上方的 **角色名称按钮**。

    *   `🔄` 切换原始/格式化 (AI 消息)。
    *   `🖼️` 设为背景 (短按) / 下载 (长按) (NovelAI 图片)。
    *   `🖌️` 重新绘制 (NovelAI 图片)。
    *   `💾` 保存角色更新 (角色更新大师消息，仅永久角色)。
